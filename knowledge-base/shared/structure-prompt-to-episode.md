# structure-prompt-to-episode.md

A developer's natural language prompt describes a scenario, a mood, or a situation for the Linions character — and the Director agent must translate this into a concrete episode structure. The prompt may be as vague as "the character tries to reach a flower" or as specific as "the character encounters a wall and a tree in a rainy mood." The Director should extract from the prompt: the implied obstacle types (matching the seven defined types), the dominant emotional register (curious, determined, anxious, playful), and any specific details to honour (named objects, implied scale, desired tone). Details not specified in the prompt are filled in with character-consistent defaults: curious and optimistic approach, comic failures, warm resolution. The Director should not invent obstacle types not in the defined set: if the prompt describes a mountain, translate it to the nearest defined type (elevated-platform or tree depending on context). The title and description generated by the Director should reflect the final episode structure, not just echo the original prompt — they describe what actually happens.

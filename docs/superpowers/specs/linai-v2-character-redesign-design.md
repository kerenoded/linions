# Linai v2 — Ecto-Cloud Character Redesign

> Status: draft

## Problem

Linai v1 is a humanoid character with legs, arms, eyelashes, dress, etc. The legs look unnatural when animated — they don't move convincingly during walking sequences. Rather than fixing leg animation, we're replacing the entire character with an energy creature that doesn't need legs at all.

## Decision

Replace Linai v1 (humanoid) with Linai v2 (ecto-cloud): a translucent ghostly ectoplasm cloud creature inspired by Ghostbusters ghosts. The character floats and flows instead of walking. Emotion is expressed through eyes + internal energy patterns (hearts, lightning, sparkles, etc.) instead of arms, eyelashes, and body posture.

The name "Linai" stays. The project names (Linions, Linoi) stay unchanged.

## Source SVG

The approved base SVG is `frontend/public/ecto-cloud.raw.svg`, generated by the DrawingAgent. Alternative color/shape variants (warm, puffy, nebula) are archived in `frontend/public/characters-v2/` for potential future use but are **out of scope** for this change.

## 1. New SVG Template

**File**: `frontend/public/linai-template.svg` (replaces existing)

**Source**: Rename `frontend/public/ecto-cloud.raw.svg` and restructure its element IDs.

### 1.1 New Part IDs

The v2 template has fewer, broader part groups:

| ID | What it contains | Animatable properties |
|---|---|---|
| `linai-body` | Main cloud shape layers, puffy bumps, outer glow, depth ellipses, inner glow | transform (translate, scale), opacity, fill |
| `linai-eye-left` | Left eye: sclera, iris, pupil, highlights | transform (translate, scale), opacity |
| `linai-eye-right` | Right eye: sclera, iris, pupil, highlights | transform (translate, scale), opacity |
| `linai-mouth` | Small smile/expression curve | d (path morph), opacity |
| `linai-inner-patterns` | Swirling energy wisps + contextual emotion symbols | opacity, fill, d (path morph) |
| `linai-particles` | Floating glowing dots inside the body | transform (translate), opacity |
| `linai-trails` | Bottom vapor trails that dissolve downward | transform (translate, scale), opacity |

**Removed IDs** (no equivalent in v2):
- `linai-head`, `linai-neck` — no separate head/neck, the cloud is one body
- `linai-eyelashes-*` — no eyelashes
- `linai-nose` — no nose
- `linai-arm-left`, `linai-arm-right` — no arms
- `linai-leg-left-group`, `linai-leg-right-group`, `linai-feet-*` — no legs/feet
- `linai-dress-heart` — no dress; the heart concept moves to `linai-inner-patterns` (hearts appear inside the cloud when in-love)
- `linai-eyeboll-left`, `linai-eyeboll-right` — folded into eye groups
- `linai-pupil-left`, `linai-pupil-right` — folded into eye groups
- `linai-eye-highlight-left`, `linai-eye-highlight-right` — folded into eye groups

### 1.2 SVG Restructuring

**Critical**: The existing `pipeline/media/linai_template.py` dynamically extracts part IDs by parsing the SVG and finding all IDs inside the `<g id="linai">` top-level group. It also injects the SVG into prompts via `{linai_template_svg}`. This means once the SVG file is correctly structured, `{linai_part_ids_json}` and `{linai_template_svg}` in Animator and Renderer prompts auto-update — no code changes needed for prompt placeholder injection.

The raw SVG from the DrawingAgent has `character-body`, `character-eye-left`, `character-eye-right`. The restructuring task:

1. Add a top-level `<g id="linai">` wrapper around all content (required by `linai_template.py` parser)
2. Rename `character-body` to `linai-body`
3. Rename `character-eye-left` to `linai-eye-left`
4. Rename `character-eye-right` to `linai-eye-right`
5. Wrap the mouth path in `<g id="linai-mouth">`
6. Wrap swirl paths in `<g id="linai-inner-patterns">`
7. Wrap floating particle circles in `<g id="linai-particles">`
8. Wrap vapor trail paths + dissolving ellipses in `<g id="linai-trails">`
9. Keep `viewBox="0 0 200 200"`

## 2. Expression System

Eyes are the primary expression tool. Internal patterns are the secondary layer.

### 2.1 Eye Expressions (7 states)

| State | Eyes | Pupils |
|---|---|---|
| neutral | Normal size, centered | Normal size, centered |
| happy | Slight squint (smaller ry) | Larger, brighter glow |
| sad | Droop downward (translate y+), slightly larger | Smaller, dimmer |
| angry | Narrow (smaller ry, slightly smaller rx) | Shrink to intense dots |
| scared | Wide open (larger ry) | Tiny, darting |
| triumphant | Wide and bright | Large, maximum glow |
| in-love | Soft squint, slightly upward | Heart-shaped or replaced by hearts |

### 2.2 Internal Pattern Emotions

The `linai-inner-patterns` group content changes per emotion. The Renderer replaces/morphs the inner SVG content:

| Emotion | Pattern inside body |
|---|---|
| neutral | Calm swirling wisps (default from template) |
| happy | Bright sparkle stars, fast-moving particles |
| sad | Slow drooping wisps, particles sink downward |
| angry | Lightning arcs / jagged energy bolts, red-tinted particles |
| scared | Particles cluster tightly toward center, swirls contract |
| triumphant | Starburst patterns, particles explode outward |
| in-love | Small hearts float upward inside the body |
| thinking/confused | Question marks, small gear shapes, or pencil shapes |

### 2.3 Color Shifting (subtle, via CSS fill transitions)

The body layers can shift color to reinforce emotion:
- **Default**: cyan-teal (#44ddee, #88eeff, #aaf4ff)
- **Happy/triumphant**: shifts warmer (#88eeff → #aaffcc)
- **Angry**: shifts toward orange-red (#ff8844, #ee5533)
- **Sad**: shifts toward muted blue-grey (#7799aa)
- **Scared**: flickering opacity pulses
- **In-love**: shifts toward warm pink (#ffaacc, #ff88bb)

## 3. Action System

Old humanoid actions are replaced with energy-creature equivalents.

| Old Action | New Action | Visual Description |
|---|---|---|
| walk | float | Body glides horizontally, gentle vertical bob, trails stream behind |
| stop | hover | Body bobs gently in place, trails settle beneath |
| jump | surge | Body compresses then shoots upward, trails flare outward |
| sit | settle | Body sinks lower, flattens slightly, trails pool on ground |
| react | pulse | Body flashes brighter, quick size pulse, particles scatter |
| fall | dissolve | Body stretches downward, becomes more transparent, trails scatter |
| celebrate | radiate | Body expands, brightens, particles spiral outward |

**Note**: The action field in keyframes is free-text (natural language), not an enum. The Animator and Renderer use the action string descriptively. This table defines the *vocabulary* the Director/Animator should prefer, not hard constraints. The Renderer interprets the text and maps it to animation.

## 4. Director Agent Updates

The Director must know Linai's new physical form to write valid story beats.

### 4.1 What the Director must NOT describe

- Linai walking, stepping, striding (she floats)
- Linai grabbing, holding, carrying, picking up (no hands/arms)
- Linai kicking, stomping (no legs/feet)
- Linai waving, pointing, gesturing with hands (no arms)
- Linai climbing, crawling (no limbs)
- Any reference to dress, hair, eyelashes

### 4.2 What the Director SHOULD describe

- Linai floating toward, drifting past, streaming over
- Linai surging upward, dissolving downward
- Linai pulsing with energy, glowing brighter/dimmer
- Linai's internal patterns changing (hearts appearing, lightning crackling)
- Linai's eyes widening, narrowing, looking in a direction
- Linai's body expanding, contracting, stretching
- Linai's color shifting to express mood
- Linai's vapor trails reacting (flaring, drooping, scattering)

### 4.3 Director Prompt Update

**File**: `pipeline/agents/director/prompt.txt`

Add a character-form section to the Director prompt that describes Linai as an ecto-cloud energy creature. The block must state:
- Linai is a translucent floating ectoplasm cloud, not a humanoid
- She has no arms, legs, hands, feet, hair, or dress
- She expresses through: eyes (size, shape, glow), internal energy patterns (hearts, sparks, lightning), body shape (expand/contract/stretch), color shifting, and vapor trails
- She moves by floating, surging, settling, dissolving — never walking, climbing, or grabbing
- Obstacle interactions must use ecto-cloud verbs (see section 4.4)

### 4.4 Obstacle Interactions Rethought

Examples of how obstacle encounters change:

| Obstacle | Old (humanoid) | New (ecto-cloud) |
|---|---|---|
| wall | Linai climbs over / breaks through | Linai surges over / phases through / squeezes through a crack |
| hole | Linai jumps across / falls in | Linai floats across / dissolves into the hole |
| puddle | Linai steps through / splashes | Linai floats over / absorbs the water and changes color |
| tree | Linai climbs / goes around | Linai flows around / drifts through branches |
| elevated-platform | Linai jumps up / climbs | Linai surges upward to the platform |
| bird | Linai ducks / waves | Linai pulses defensively / shrinks away |
